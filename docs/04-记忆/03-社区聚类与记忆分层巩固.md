# 社区聚类（LPA）与记忆分层巩固 — 设计与面试

> 把语义相关的记忆实体自动聚成「主题社区」；给记忆加「轻重冷暖」属性，短期记忆达标后巩固为长期。让记忆从扁平事实集合变成有结构、有动力学的活体。
> 对应能力域：**记忆 / 图算法 / 记忆动力学**。代码：`core/memory/clustering/label_propagation.py`、`core/memory/consolidation/consolidator.py`。

---

## 0. 能力定位（对应招聘要求）

- 对应 JD：**「图算法 / 社区发现」「长短期记忆设计」「记忆动力学」**。
- 角色：聚类给记忆图谱**主题结构**（用于可视化、组织），分层巩固给记忆**价值梯度**（重要稳定的优先浮现）。

## 1. 解决什么问题

- **聚类**：图谱实体越来越多，散落无组织。想自动把相关的（工作相关、家人相关、某爱好相关）聚成主题社区，便于浏览和理解。
- **分层巩固**：所有记忆一视同仁不合理——有的重要稳定（职业）、有的琐碎一次性。想让记忆有「轻重」，重要的、常被用到的优先浮现，又**不遗忘**任何记忆。

## 3. 核心设计与实现（后端）

### 3.1 社区聚类：标签传播 LPA（`label_propagation.py`）

**LPA（Label Propagation Algorithm）** 思想是「物以类聚」：
1. **初始化**：每个实体各自一个独立标签（社区）。
2. **迭代投票**：每个实体看自己的邻居，按**加权投票**采用得票最高的社区标签。权重 = **语义相似度（name_embedding 余弦）× 0.6 + 关系连接 × 0.4**——既看语义近不近，也看有没有关系边连着。最多 10 轮，没有标签变化就收敛。
3. **合并相似社区**：算每个社区的**平均向量**，两社区平均向量余弦 > 0.85 就合并（保留成员多的一方，用并查集 root 管理合并链）。
4. **生成元数据**：对每个社区用 LLM 起中文名 + 一句话摘要（没配模型则成员名拼接兜底）。

> 面试一句话：用标签传播 LPA——每个实体初始各一个标签，按邻居加权投票（语义相似度 0.6 + 关系连接 0.4）迭代采用得票最高的标签直到收敛，再合并平均向量很近的社区，最后 LLM 给社区起名。优点是快、线性复杂度、无需预设社区数。

### 3.2 双触发：增量 + 定时

- **增量**：每次萃取写图后，对**新实体**跑 `incremental_update`——新实体按邻居投票归入已有社区或新建（只动局部，快、保实时）。
- **定时全量**：Celery beat 每天凌晨跑 `full_clustering` 全量重聚类（修正增量累积的偏差、合并该合并的社区，保质量）。增量保实时、全量保质量。

### 3.3 记忆动力学属性

给图节点加属性，让记忆有「轻重冷暖」：
- **通用**（实体/陈述/关系）：`importance` 重要度（萃取时 LLM 评分）、`confidence` 置信度、`access_count` 被检索命中次数（检索回写）、`memory_layer` 层级（short_term/long_term）。
- **实体专属**：`mention_count` 被提及次数（每次萃取命中 +1）、`core_facts` 核心事实、`traits` 特质标签（巩固时 LLM 产出）。
- **来源双轨**：① LLM 萃取时直接评 importance/confidence；② 运行中累积 access_count（检索命中）/ mention_count（重复提及）。

### 3.4 短期→长期巩固（`consolidator.py`，每天 4:00）

新实体默认 short_term。定时巩固把满足**任一**条件的短期记忆提升为 long_term（**只升不降**）：
- `access_count ≥ 阈值`（经常被检索复用）；
- `importance ≥ 阈值`（本身重要）；
- `mention_count ≥ 阈值` 且已存在满 N 小时（反复提及 + 沉淀过一段时间）。

「任一达标即升」捕捉了记忆变重要的不同信号（被用、本身重要、被反复提起）。

### 3.5 画像增强（巩固的第二步）

提升后，对 **top-K 高频长期实体**做画像增强：用 LLM 汇总该实体的关联陈述（≥2 条才做），产出 `core_facts`（核心事实）和 `traits`（特质标签）回写。比如「用户」这个高频实体，汇总出「核心事实：在腾讯做后端、养狗多多；特质：爱学习、技术控」。让高价值实体有浓缩画像。

### 3.6 检索如何消费动力学（闭环）

记忆检索融合分 = `0.55·向量 + 0.30·全文 + 0.15·importance`，长期记忆额外 +0.05；命中回写 access_count。于是「越重要/越常被检索 → 排序越靠前 → 越易被命中 → 越快被巩固为长期」形成**高价值记忆浮现**的正反馈。这就把「聚类的结构」「巩固的价值梯度」「检索的排序」串成一个闭环。

### 3.7 「记忆不遗忘」原则

全程**只升不降、不删不衰减**——这是本项目刻意区别于 ACT-R / 艾宾浩斯遗忘引擎的设计。记忆层级单向提升、重要度/访问次数只增。因为个人记忆助手的核心价值是**长期留存用户信息**，遗忘曲线适合工作记忆/缓存淘汰，与产品定位相悖。分层的意义不是淘汰旧记忆，而是让稳定记忆检索时优先 + 触发画像增强。

## 4. 关键设计取舍

| 决策点 | 选了什么 | 备选 | 为什么 |
|--------|---------|------|--------|
| 聚类算法 | LPA 标签传播 | K-means / Louvain / Leiden | 快、线性、无需预设社区数；投票加语义+关系比纯结构更贴合 |
| 聚类触发 | 增量 + 定时全量 | 只全量 / 只增量 | 增量保实时、全量保质量 |
| 社区命名 | LLM 起名 + 兜底拼接 | 纯算法标签 | LLM 名可读，没配模型也能用 |
| 记忆梯度 | 动力学属性 + 短长分层 | 一视同仁 | 高价值记忆优先浮现 |
| 巩固条件 | access/importance/mention 任一 | 全部满足 / 单一 | 捕捉记忆变重要的不同信号 |
| 遗忘 | 不做（只升不降） | ACT-R 衰减 | 个人记忆助手要长期留存，遗忘与定位相悖 |

## 5. 踩坑与解决

- **LPA 结果不稳定**（标准 LPA 因更新顺序随机可能每次结果不同）：解法：加权投票（语义+关系）降随机性 + 最大迭代上限 + 社区合并收敛；增量只动新实体不全量重算。
- **社区无限碎裂**：解法：平均向量余弦 > 0.85 合并相似社区。
- **画像增强对陈述太少的实体没意义**：解法：陈述 < 2 条跳过。
- **正反馈马太效应淹没新记忆**：解法：importance 只占 0.15 权重不主导，新记忆仍能靠向量/全文相关度召回；巩固只升不挤占新记忆召回机会。

## 6. 面试问答

**Q1（核心）：社区聚类用什么算法？为什么？**
标签传播 LPA：每个实体初始各一标签，按邻居加权投票（语义相似度 0.6 + 关系连接 0.4）迭代采用得票最高标签直到收敛，再合并相似社区，LLM 起名。优点快、线性复杂度、无需预设社区数，比纯结构 LPA 多了语义权重更贴合。

**Q2（工程）：LPA 不稳定怎么办？**
标准 LPA 因更新顺序随机可能不稳定。我用加权投票降随机性、设最大迭代和社区合并收敛、增量只动新实体不全量重算。要更稳可换 Leiden/Louvain。

**Q3（设计）：记忆分层怎么实现？为什么不做遗忘？**
每个实体/陈述带 memory_layer，定时巩固按「access_count/importance/mention 任一达标」把短期提升为长期，只升不降。不做遗忘是因为个人记忆助手核心价值是长期留存用户信息，遗忘曲线适合工作记忆/缓存淘汰，与定位相悖。分层意义是让稳定记忆检索优先、触发画像增强，不是淘汰。

**Q4（原理）：巩固为什么是「任一达标」？**
记忆变重要有不同信号：被反复检索用到（access）、本身就重要（importance）、被反复提起且沉淀过（mention + 时长）。任一达标都说明这条记忆有价值，所以「任一」而非「全部」。

**Q5（闭环）：动力学属性怎么参与排序？会正反馈失控吗？**
检索融合分 = 0.55 向量 + 0.30 全文 + 0.15 重要度，长期 +0.05，命中回写 access。确实是正反馈（越重要越易被检索越快巩固），这正是要的高价值浮现。防马太效应：importance 只占 0.15 不主导，新记忆仍能靠相关度召回。

**Q6（增量 vs 全量）：两种聚类触发怎么配合？**
萃取写图后增量触发（新实体按邻居投票归入，快、只动局部）；beat 每天凌晨全量重聚类兜底（修正增量偏差、合并社区）。增量保实时、全量保质量。

## 7. 相关论文 / 概念

**① 社区发现（Community Detection）算法谱系**
图上找「内部连接密、外部连接疏」的节点群：
- **模块度优化**：**Louvain（Blondel et al. 2008）** 贪心优化模块度（modularity），快且效果好，被广泛使用 → **Leiden（Traag et al. 2019）** 修复了 Louvain 可能产生「不连通社区」的缺陷，是当前推荐。
- **标签传播 LPA（Raghavan et al. 2007）**：每个节点采用邻居中最多的标签迭代至收敛。**近线性复杂度、无需预设社区数、极快**，缺点是结果不够稳定。本项目用 LPA（加语义+关系加权降随机性），看重它快和无需预设社区数；列 Leiden/Louvain 为可优化方向。

**② 记忆的认知模型与「遗忘」**
- **艾宾浩斯遗忘曲线（Ebbinghaus 1885）**：记忆随时间指数衰减，复习可强化。
- **ACT-R（Anderson）认知架构**：记忆激活值随时间衰减、随使用强化，是很多「AI 记忆遗忘」设计的理论来源。
- **短期 / 长期记忆**：认知心理学的记忆分层。
- 本项目**刻意不做遗忘**——借鉴了「分层 + 使用强化（access/mention）」的思想，但去掉了衰减/淘汰，因为个人记忆助手要的是「不忘」。这是「按产品定位裁剪学术模型」的典型决策。

**③ 知识图谱嵌入与社区**
社区平均向量、实体 name_embedding 投票，用到了「实体可嵌入向量空间且语义可比」的思想（TransE 等 KG embedding 的基础假设）。

> 一句话脉络：社区发现从 Louvain（模块度）到 Leiden（修连通性缺陷）到 LPA（极快、无需预设数），本项目用加权 LPA；记忆动力学借鉴艾宾浩斯/ACT-R 的「分层 + 使用强化」，但按「个人助手要不忘」的定位去掉了遗忘衰减。
