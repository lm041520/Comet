## 一、项目整体（开场必问）

**Q：一句话介绍你的项目。**
多用户个人 AI 知识库 + 记忆助手：RAG 知识库 + Neo4j 记忆图谱 + LangChain Agent 自主编排工具问答，进阶有深度研究、定时任务、多 Agent 群聊。FastAPI + React，四存储 + Celery。→ `00-总览`

**Q：为什么用四种存储？**
PG 结构化业务（事务）、ES 向量+BM25 混合检索、Neo4j 实体关系图（多跳遍历）、Redis 缓存+Celery broker。各司其职。→ `00-总览`

**Q：跨存储一致性怎么保证？**
无强一致需求，最终一致：PG 事务保元数据，异步任务出错置 failed，ES/Neo4j 用幂等（删旧重写/MERGE）保可重试。→ `00-总览`

**Q：架构怎么分层？**
controller→service→repository→model/db 单向；横切能力放 core 按子系统分目录，子系统内按流水线阶段拆。→ `00-总览`

## 二、RAG（检索）

**Q：什么是混合检索？为什么向量和 BM25 都要？**
向量懂语义（同义召回）、BM25 精确匹配关键词，互补。归一化加权融合（向量 0.6 + BM25 0.4）。→ `02-RAG/混合检索与rerank`

**Q：rerank 和向量召回区别？**
向量是 bi-encoder（分开编码，快粗），rerank 是 cross-encoder（拼一起算，慢准）。召回粗筛 + rerank 精排两段式。→ `02-RAG/混合检索与rerank`

**Q：父子分块解决什么？**
small-to-big：小子块向量召回（准），命中返回大父块（上下文全）。→ `02-RAG/分块与中文分词`

**Q：中文检索怎么处理分词？**
装 ES IK 插件，写入 ik_max_word 细切、查询 ik_smart 粗切，否则按单字切 BM25 差。→ `02-RAG/分块与中文分词`

**Q：引用怎么保证真实？**
不让 LLM 自标（会编），检索命中时收集真实来源，SSE 单独事件吐给前端。→ `02-RAG/引用溯源与多知识库`

**Q：全局搜索为什么要余弦门控？**
向量 kNN 永远返 top-k，没相关内容也塞最近邻。门控只展示余弦≥阈值的。问答偏召回、搜索偏精确。→ `02-RAG/全局搜索与语义门控`

**Q：图片怎么进知识库检索？**
多模态模型把图转描述文本再向量化，「以文搜图」。→ `02-RAG/多模态识别与文档预览`

## 三、Agent（编排）

**Q：Agent 怎么编排工具？**
方案 B 双路径：强模型 function calling 工具循环、弱模型 ReAct prompt 模拟。检索做成工具让 LLM 自主调。→ `03-Agent/FunctionCalling与ReAct双路径`

**Q：Function Calling 和 ReAct 区别？**
前者模型原生结构化调用（可靠，需支持）、后者 prompt+文本解析（兼容任何模型，脆）。都做按模型能力路由。→ `03-Agent/FunctionCalling与ReAct双路径`

**Q：怎么防工具死循环？**
最大迭代 5 轮 + 同轮 call_cache 缓存相同调用。→ `03-Agent/FunctionCalling与ReAct双路径`

**Q：MCP 是什么？怎么接入的？**
AI 工具标准协议，用官方 langchain-mcp-adapters 把远程 MCP server 工具运行时拉成 LangChain 工具注入，不改代码扩展工具。→ `03-Agent/工具注册中心与MCP接入`

**Q：MCP 性能怎么优化？**
工具清单 TTL 缓存（按 server 指纹失效）+ 持久会话（多次调用复用连接）/ 无状态（单聊只在真调用时连）。→ `03-Agent/工具注册中心与MCP接入`

**Q：角色卡和技能区别？**
角色卡定「我是谁」（人设 prompt），技能定「干什么任务」（任务 prompt+few-shot+工具白名单），叠加组合。→ `03-Agent/角色卡(人格)与提示词优化器`

## 四、记忆（知识图谱）

**Q：怎么从对话萃取记忆？**
先切原子陈述 + 指代消解，再 LLM 按受控词表（13 实体+13 谓词）抽三元组，异步执行。→ `04-记忆/三元组萃取与四层溯源图谱`

**Q：实体去重怎么做？为什么会有重复用户节点？**
规则初筛 + LLM 精判，两层（批内+图侧），同名快路径直接复用。重复节点根因是早期同名也问 LLM、非确定性，加同名快路径修复。→ `04-记忆/实体去重与图混合检索`

**Q：记忆检索和 RAG 区别？**
记忆多了重要度加权 + 一跳邻居关系遍历（带出关联事实），RAG 只返文本块。→ `04-记忆/实体去重与图混合检索`

**Q：为什么用图数据库存记忆？**
记忆是实体关系结构，多跳遍历图库高效，关系库要反复自连接。→ `04-记忆/三元组萃取与四层溯源图谱`

**Q：社区聚类用什么算法？**
LPA 标签传播，加权投票（语义 0.6+关系 0.4）迭代收敛，增量+定时双触发。→ `04-记忆/社区聚类与记忆分层巩固`

**Q：记忆分层/巩固怎么做？为什么不遗忘？**
动力学属性 + 短长分层，access/importance/mention 任一达标提升为长期，只升不降。个人助手要长期留存，遗忘与定位相悖。→ `04-记忆/社区聚类与记忆分层巩固`

**Q：反思引擎干嘛？**
回看高价值实体让 LLM 归纳高层洞察 Insight（源自 Generative Agents 记忆流+反思）。→ `04-记忆/反思引擎(Insight)`

**Q：主动召回怎么不拖慢对话？**
用户级温热缓存每轮 0 阻塞注入，非寒暄消息后台异步刷新；3.5s 超时降级。→ `04-记忆/主动召回与跨会话上下文`

## 五、多 Agent / 复杂任务

**Q：深度研究怎么实现？**
八步流水线：规划→四源检索→逐源提炼→反思补搜→大纲整理→分章节写作→汇总→引用映射。对标 GPT Researcher/STORM。→ `05/深度研究引擎`

**Q：定时任务调度的健壮性？**
心跳/执行拆队列、FOR UPDATE SKIP LOCKED 原子认领、Redis 锁防重叠、asyncio.wait_for 硬超时、先推进再派发。→ `05/定时主动任务调度`

**Q：多 Agent 群聊怎么调度？**
主持人 LLM 决定发言顺序（同 AutoGen GroupChatManager），文本 transcript 承载多方上下文 + 一轮内动态累加实现接话。→ `05/多Agent群聊`

## 六、工程化

**Q：为什么用 Celery？怎么拆队列？**
慢操作异步化不卡接口。按域拆 parse/memory/beat/research，重活不堵轻活（Bulkhead）。→ `06/Celery异步多队列`

**Q：Celery 跑异步代码踩过什么坑？**
全局 DB 池绑主循环、任务用新循环会报错，任务级独立引擎+NullPool。→ `06/Celery异步多队列`

**Q：API Key 怎么存？密码呢？**
Key 用 Fernet 对称加密（可逆，调用要还原）+ 掩码；密码 bcrypt 哈希（只验证不还原）。→ `01/配置加密` `06/账号鉴权`

**Q：怎么防 SSRF？**
用户可控 URL（网页导入/MCP）校验解析出的所有 IP，内网/保留段拒绝。→ `06/账号鉴权与安全`

**Q：多用户数据怎么隔离？**
所有表带 user_id，所有查询（PG/ES/Neo4j/向量）强制带 user_id 过滤，做成固定模式。→ `06/账号鉴权与安全`

**Q：SSE 部署为什么不流式？**
Nginx 默认缓冲响应，加 X-Accel-Buffering: no 关缓冲。→ `01/SSE流式输出` `06/部署踩坑集`

**Q：分享公开页不鉴权怎么安全？**
快照式 + 不可猜 token + 只暴露快照 + 头像图片转 data URL。→ `06/分享与导出`

## 七、亮点 / 加分项（主动讲）

- **方案 B 双路径**：兼容强弱模型（function calling / ReAct 降级）。
- **两层实体去重 + 同名快路径**：解决 LLM 非确定性导致的重复节点。
- **记忆动力学闭环**：萃取→巩固→反思→主动召回，高价值记忆浮现。
- **深度研究 v2**：对标 GPT Researcher 逐源提炼、STORM 大纲优先。
- **调度健壮性五点加固**：拆队列/原子认领/锁/超时/先推进。
- **真实性能优化**：MCP 握手消除、主动召回缓存、首屏不等 LLM。
- **诚实的踩坑与取舍**：ES 动态映射 reindex、Celery 事件循环、真人模式调优回滚——能讲清「为什么这么选、踩了什么坑」。

## 八、可能被问倒的（提前想好）

- **评估（eval）**：目前没有系统化 RAG/Agent 自动评估——诚实说是可优化方向（可上 RAGAS、人工标注集）。
- **并发规模**：个人项目数据量小，单索引/单机够用；规模化要分片、读写分离、向量库专用（Milvus）。
- **成本**：没做 token 成本统计/限流——可优化方向。
