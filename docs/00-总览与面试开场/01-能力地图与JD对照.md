# 能力地图与 JD 对照 — Comet 覆盖了 Agent 岗哪些要求

> 把 Comet 的功能「翻译」成 Agent 开发岗的招聘语言，逐条对照。面试前看这张表，知道哪个项目点对应 JD 的哪条要求、去哪篇复习。

---

## 一、Agent 开发岗的能力地图（招聘要求综合）

综合多个大模型 / Agent 开发岗 JD，核心能力域如下（这也是本套文档 01~07 的分类依据）：

| 能力域 | JD 常见表述 |
|--------|------------|
| LLM 应用工程 | 主流大模型 API 接入、prompt 工程、流式输出、token/上下文管理 |
| RAG | 向量检索、混合检索、rerank、分块、中文分词、向量库、引用溯源、GraphRAG |
| Agent 核心 | Function Calling、ReAct、工具编排、多步规划、反思、MCP、Agentic RAG |
| 记忆 | 知识图谱、长短期记忆、动态记忆、按需召回、意图识别 |
| 多 Agent | 多智能体协作、Agent 调度编排（类 STORM / AutoGen / GPT Researcher） |
| 工程化 | Python/FastAPI/异步、Celery、向量库、部署、可观测、评估、安全 |
| 加分 | 多模态、情感计算、推荐、复杂任务自动化 |

## 二、逐条对照（JD 要求 → Comet 实现 → 复习文档）

## 三、按岗位侧重选讲（投不同岗时的重点）

- **偏 Agent 岗**：重点讲 03（双路径 + MCP）、05（深度研究 + 群聊）、04（记忆按需召回）。
- **偏 RAG / 知识工程岗**：重点讲 02（混合检索 + rerank + 多库）、04（知识图谱 + GraphRAG 思想）。
- **偏 LLM 应用 / 全栈岗**：重点讲 01（多模型适配 + SSE）、06（工程化 + 部署）、整体架构。
- **偏算法岗**：补强 04（图算法 LPA、记忆动力学）、07（情绪计算）、各篇的「相关论文」节。

## 四、覆盖度自评（诚实）

- **强覆盖**：RAG 全链路、Agent 双路径编排、知识图谱记忆、多 Agent 编排、工程化——这些是项目主体，能讲深、有取舍、有踩坑。
- **有但偏轻**：多模态（以文搜图，未做图像 embedding）、情感计算（应用层，未训模型）。
- **未覆盖（诚实说 + 给方向）**：系统化评估（eval，可上 RAGAS）、大规模性能（分片/专用向量库）、成本统计/限流、模型微调（项目用的是 prompt + 编排，未做 SFT/RLHF）。被问到时坦诚「这个项目没做，但我了解方向是 X」。
